Section 01
Quick Start — 5 Minutes

If TaxBot files are already in C:\MyERP\, start here.

Open PowerShell (normal user)

Press Win + R → type powershell → Enter. Do not run as Administrator.

cd C:\MyERP
Run setup (first time only)
python taxbot.py setup
✅
You should see "Packages installed", "Scripts dir saved to user PATH permanently", and "Google API libraries loaded ✓". Gmail is optional — local folder drop works without it.
Load demo data (recommended)
python taxbot.py demo

Loads 4 real Kenyan clients with sample receipts so you can explore the dashboard immediately.

Open the dashboard

Double-click dashboard.bat, or:

python taxbot.py dashboard

Browser opens at http://localhost:8501

Start the receipt watcher

Double-click watch.bat, or in a second terminal:

python taxbot.py watch

Drop receipts into any client's inbox\ folder — processed in ~2 seconds.

Section 03
Python PATH Fix

If you see "python is not recognized" or "streamlit.exe is installed in …Scripts which is not on PATH", use these fixes. Common with Python 3.14 (winget install) and corporate Windows machines.

⚠️
What happened: Python 3.14 installed via winget uses the path C:\Users\…\AppData\Local\Python\pythoncore-3.14-64\. Its Scripts folder (where streamlit.exe lives) is never added to PATH automatically.
🩹 Fix A — Double-click FIX.bat (easiest)
In File Explorer, go to C:\MyERP\ and double-click FIX.bat. It detects your Python (including pythoncore-3.14-64 style paths), adds Scripts to PATH permanently, installs all packages, and creates launcher .bat files.
🔧 Fix B — Re-run setup
python taxbot.py setup
Setup auto-detects the Scripts dir, adds it to the current session, and saves it permanently via a PowerShell .ps1 file. Open a new PowerShell window afterwards for the change to take effect.
⚡ Fix C — Use launcher .bat files (always work)
After FIX.bat runs, four launchers are created that use the exact Python path — no PATH dependency:
FileWhat it does
watch.batStarts receipt watcher + Gmail scan
dashboard.batOpens Streamlit at localhost:8501
start.batRun any command: start.bat status
runqa.batRuns the full QA suite
💡
PATH changes only apply to new terminal windows. After running setup or FIX.bat, open a fresh PowerShell before testing streamlit --version.
⚠️
PowerShell: username with spaces (e.g. “John Kamau”)
If pip prints a long path and you copy-paste it directly, PowerShell throws CommandNotFoundException because the space splits the path.
Fix — wrap in quotes and prefix with &: 
# Wrong (space breaks the path)
C:\Users\John Kamau\AppData\...\python.exe -m pip install ...

# Correct
& "C:\Users\John Kamau\AppData\Local\Python\pythoncore-3.14-64\python.exe" -m pip install --upgrade pip
& "C:\Users\John Kamau\AppData\Local\Python\pythoncore-3.14-64\python.exe" -m pip install python-docx openpyxl pymupdf

Three ingestion paths. All converge at the same AI extraction pipeline.

📄 Supported Formats
FormatProcessingRequires
.jpg .jpeg .png .bmp .gif .tiff .webpTesseract OCR → Qwen AITesseract installed
.pdfPyMuPDF text extraction → Qwen AINothing extra
.txt .csvDirect read → Qwen AI or regex fallbackNothing extra
🔄 Processing Pipeline (per file)
  1. File detected in inbox\
  2. MD5 hash checked — duplicates skipped
  3. OCR or text extraction
  4. AI extracts: vendor, date, amount, VAT, category, payment method, confidence
  5. Row written to Excel (status OK or REVIEW)
  6. Double-entry journal entry posted to Ledger sheet
  7. Audit log entry created (AuditLog sheet)
  8. File moved to processed\ or failed\

📅 Month-End Checklist (per client, ~15 min)
  1. Run the full sequence in one command: python taxbot.py monthly <id> --month YYYY-MM — this does steps 2–8 below automatically. Only run individual steps if you need to redo one specific report.

Section 13
Troubleshooting
❌ "python is not recognized"

Cause: Python not in PATH. Common with Python 3.14 (winget install at pythoncore-3.14-64).

Fix: Double-click FIX.bat. It scans all known install paths including %LOCALAPPDATA%\Python\*, fixes PATH permanently, and creates launcher .bat files.

❌ "streamlit is not recognized" after install

Cause: Scripts folder not in PATH. Packages installed but tools not findable.

Fix: Run python taxbot.py setup (auto-adds Scripts dir). Or just use dashboard.bat — it uses exact path, no PATH needed.

❌ "Google API libraries not installed" after setup

Cause: Python cached GMAIL_OK=False before packages were installed. This is now fixed — setup reloads libs in the same session.

Fix: Open a new PowerShell window and run python taxbot.py setup again.

❌ "Auth returned None — check credentials file"

Checklist:

  • File is named exactly gmail_credentials.json
  • File is in C:\MyERP\ (same folder as taxbot.py)
  • Your Gmail was added to "Test users" in Google Cloud OAuth consent screen
  • Gmail API was enabled in the Cloud project
❌ Excel file locked / PermissionError
Close the workbook in Excel before running TaxBot. Windows locks files opened in Excel. TaxBot has retry logic but cannot override an active file lock.
❌ Ollama not responding
ollama list       # Check if running
ollama serve      # Start if not running
ollama pull qwen2.5:3b  # Re-pull model if missing

Note: TaxBot falls back to regex if Ollama is offline. All receipts still process — with lower accuracy on ambiguous data.

❌ "Tesseract not found"
Image OCR disabled. PDF text and .txt/.csv still work. Install from https://github.com/UB-Mannheim/tesseract/wiki to C:\Program Files\Tesseract-OCR\ then re-run setup.
