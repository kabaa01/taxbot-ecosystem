📅 Weekly Recurring Tasks — Every Client

ℹ️
These tasks run on a fixed weekly schedule. Assign one staff member per task. Every task has a TaxBot command. Print this page and pin it on the office wall.
📆
MONDAY — Data & Onboarding
9:00 AM · 45 min
  • Open TaxBot portal → check new client submissions from weekend (client_onboarding.html)
  • For each new client: python taxbot.py add-client or python taxbot.py add-individual
  • Import onboarding JSON files from disconnected sessions: python taxbot.py import-onboard <file.json>
  • Scan Gmail for new receipts: python taxbot.py scan-gmail
  • Process all pending receipts: python taxbot.py process
  • Send WhatsApp status update to new clients (use template below)
📊
TUESDAY — Reports & Analysis
9:00 AM · 60 min
  • Run monthly sequence for each business client: python taxbot.py monthly <id>
  • Generate P&L for any client that had receipts last week: python taxbot.py pl <id>
  • Check Trial Balance for any discrepancies: python taxbot.py tb <id>
  • Flag REVIEW rows in Expenses sheet for client clarification
  • Generate advisory report for clients due quarterly update
🏦
WEDNESDAY — Tax Compliance
9:00 AM · 90 min
  • KRA Deadline File VAT-3 for any client with VAT due (20th of each month)
  • KRA Deadline Remit PAYE by 9th — check all payrolls are processed
  • Generate KRA eTIMS template: python taxbot.py kra-input <id>
  • After client completes template: python taxbot.py kra-run <id>
  • Remit NSSF by 15th · SHIF by 9th · AHL by 9th
  • Check compliance-check output: python taxbot.py compliance-check
  • File NITA levy (0.5% of gross payroll) by 20th
📧
THURSDAY — Client Communication
2:00 PM · 45 min
  • Send weekly brief to all clients: python taxbot.py weekly-brief
  • Send handover package to accountant clients: python taxbot.py handover <id>
  • Follow up on any unresolved REVIEW transactions via WhatsApp/email
  • Send status update to personal tax clients awaiting return
  • Send advisory report (PDF/Excel) to clients with completed quarter
  • Log all client interactions in the Audit Log
🔍
FRIDAY — QA & Maintenance
3:00 PM · 30 min
  • Run full QA suite: python qa_full.py --report — must be 100%
  • Check error log: type C:\MyERP\logs\taxbot.log | findstr ERROR
  • Back up all client workbooks to external drive
  • Update Ollama model if new version available: ollama pull qwen2.5:3b
  • Generate billing invoice for all clients: python taxbot.py billing
  • Send billing to clients by end of month
⚡
DAILY (Every Day)
8:30 AM · 10 min
  • Start TaxBot watch mode (keep running all day): python taxbot.py watch
  • Check for any new receipts dropped in inbox folders
  • Review any URGENT client WhatsApp messages
  • Check KRA iTax portal for any notices or demand letters
  • Note: TaxBot watch mode auto-processes — you supervise, not manually push

👤 Personal Tax Clients — Individual Filers

✅
No business registration needed. These clients file once a year using their personal KRA PIN (format A000000000X). They do NOT use eTIMS as a registered business — just their personal PIN on the KRA iTax portal.

👥 Client Groups — How to Categorise

💼
Group 1: Employed (PAYE Only)
Simplest · File once a year

Salary earner with one employer. PAYE already deducted. Annual return confirms no underpayment.

  • ✓
    Needs: P9A from employer
  • ✓
    File IT1 nil return if no other income
  • ✓
    Claim: mortgage interest, pension, SHIF
  • ✓
    Fee: KES 3,000 (standard)

TaxBot python taxbot.py add-individual

🏠
Group 2: Employed + Rental Income
Medium complexity

Has salary + residential or commercial rental. Rental WHT at 10% (residential) or included in IT1 (commercial).

  • ✓
    Needs: P9A + rental receipts/agreements
  • ✓
    Residential rent
  • ✓
    Residential rent 288K-15M = 10% WHT final
  • ✓
    Deduct: agent fees, repairs, rates, depreciation
  • ✓
    Fee: KES 6,500
💡
Group 3: Freelancer / Consultant
Self-employed, no biz registration

Earns fees/consultancy without a registered company. Subject to WHT 5% at source from clients. Must file IT1 annually.

  • ✓
    Needs: WHT certificates from each client, bank statements
  • ✓
    Deduct: home office, equipment, transport, internet
  • ✓
    If earning >KES 5M: must register for VAT
  • ✓
    Consider TOT (1%) if revenue
  • ✓
    Fee: KES 8,000
📈
Group 4: Retired / Pensioner
Pension + investments

Pension income + dividends + interest. May have multiple income streams. Needs careful calculation of tax reliefs.

  • ✓
    Needs: pension statement, bank interest certificates
  • ✓
    NSSF pension payout: tax on full amount less personal relief
  • ✓
    Dividends: 5% WHT final (residents)
  • ✓
    Interest: 15% WHT final (bank savings)
  • ✓
    Fee: KES 5,500
🌍
Group 5: Multiple Employers / Complex
Most complex individual

Works for 2+ employers, or has employment + business + rentals + foreign income. Full IT1 required with all sources consolidated.

  • ✓
    Needs: P9A from ALL employers, WHT certs, rental records
  • ✓
    Consolidate all income onto one IT1 return
  • ✓
    Foreign income: declare + claim tax credit (DTAs)
  • ✓
    Director of a company: include director's fees
  • ✓
    Fee: KES 12,000–18,000
🏡
Group 6: Investor (No Employment)
Dividends, gains, interest only

No salary — lives on investment returns. Dividends (5% WHT final), interest (15% WHT final), capital gains on shares (5% CGT).

  • ✓
    Capital gains on shares: 5% CGT (sold after 3 yrs = exempt)
  • ✓
    NSE listed shares: 5% CGT
  • ✓
    Property gains: 5% CGT on gross proceeds
  • ✓
    WHT on dividends is final for residents
  • ✓
    Fee: KES 6,500–10,000

📋 Personal Tax Annual Checklist (per individual client)

  • STEP 1 — DOCUMENTS: Collect P9A (from employer), bank statements, rental receipts, WHT certificates, NSSF statements, SHIF receipts, mortgage statements
  • STEP 2 — CLIENT ONBOARD: If new client, open client_onboarding.html or taxbot_portal.html, select "Individual", complete form offline → export JSON
  • STEP 3 — REGISTER: Import JSON: python taxbot.py add-individual
  • STEP 4 — GENERATE IT1 WORKBOOK: python taxbot.py personal-return <id> → fill the Excel template with all income sources
  • STEP 5 — TAX COMPUTATION: Review calculated figures: gross tax, personal relief (KES 28,800/yr), reliefs, PAYE already paid, balance due/refund
  • STEP 6 — CLIENT REVIEW: Send draft to client for approval. Use status messaging: "We are reviewing your file and looking for every legal way to reduce your tax."
  • STEP 7 — FILE: Login to iTax (itax.kra.go.ke) → Individual → Returns → IT1 → upload or fill online → submit
  • STEP 8 — PAYMENT: If tax due, pay via KRA iTax payment gateway → M-Pesa or bank → get e-slip
  • STEP 9 — CONFIRMATION: Download acknowledgement slip from iTax → send to client → file in records
  • STEP 10 — INVOICE: Issue VAT-exempt invoice to client → receive payment

Set these up ONCE, then they run automatically. The scheduled tasks below run TaxBot commands on a fixed schedule. Your staff only review the output — they don't manually push buttons every day.

📁 weekly_taxbot.bat — Save to C:\MyERP\ and schedule
@echo off
REM ── TaxBot Weekly Automation ─────────────────────────────────
REM Run every Monday 8:30 AM via Task Scheduler

cd /d C:\MyERP

REM 1. Scan Gmail for new receipts
python taxbot.py scan-gmail >> C:\MyERP\logs\weekly.log 2>&1

REM 2. Process all pending receipts
python taxbot.py process >> C:\MyERP\logs\weekly.log 2>&1

REM 3. Generate weekly brief for all clients
python taxbot.py weekly-brief >> C:\MyERP\logs\weekly.log 2>&1

REM 4. Run QA check
python qa_full.py --report >> C:\MyERP\logs\weekly.log 2>&1

REM 5. Check for compliance issues
python taxbot.py compliance-check >> C:\MyERP\logs\weekly.log 2>&1

REM Done — review weekly.log for any errors
echo Weekly run complete: %date% %time% >> C:\MyERP\logs\weekly.log
💻 Set up Task Scheduler (run once as Administrator)
REM Create weekly scheduled task
schtasks /create /tn "TaxBot Weekly" /tr "C:\MyERP\weekly_taxbot.bat" /sc WEEKLY /d MON /st 08:30 /ru SYSTEM /f

REM Create daily watch task (runs continuously)
schtasks /create /tn "TaxBot Daily Watch" /tr "python C:\MyERP\taxbot.py watch" /sc DAILY /st 08:00 /ru SYSTEM /f

REM Verify tasks are created
schtasks /query /tn "TaxBot Weekly"

⚙️ Fix 2 — Cash Flow Direction Toggle

The client taps 💸 Money OUT (red, default) or 💰 Money IN (green) before filling any other field. The two controls are bidirectionally linked — changing the doc type syncs the toggle and vice versa. The email subject automatically changes:

  • 💸
    Expense: subject reads [EXPENSE] Company | KES X | Date | Vendor
  • 💰
    Income: subject reads [INCOME] Company | KES X | Date | Vendor

The email body includes the full ledger hint: "DR Bank / CR Revenue" — the accountant never has to guess.

⚙️ Fix 3 — Known Revenue Sources (Manage in PWA Setup Tab)

The accountant pre-loads a list of vendor names that are always income. When a client types a matching vendor name, the PWA automatically flips to Money IN and selects Settlement Statement. Default list includes:

Karirana Estates KTDA Maramba Limuru Tea James Finlay Eastern Produce

To add a client's income source: Open the PWA → Setup tab → scroll to "Known Income Sources" → type the vendor name → tap Add. The source is stored locally in the client's browser.

Accountant action: When onboarding a farming or cooperative client, always ask "Who pays you for your produce?" and add those names to their Known Income Sources immediately.

📥 income_inbox Folder — Accountant-Side Safety Net

ℹ️
For clients who cannot use the PWA classification (e.g. they just forward PDFs without classifying), use the income_inbox folder as a second line of defence.
  • Create folder: C:\MyERP\clients\<id>\income_inbox\
  • Drop all factory statements, KTDA payments, cooperative payouts here — NOT in the regular inbox\
  • Run: python taxbot.py process-income <id> — posts as DR Bank / CR Revenue automatically
  • Regular inbox\ continues to be treated as expenses
  • Add known revenue vendors to TaxBot registry: python taxbot.py add-revenue-source <id> "Karirana Estates"
